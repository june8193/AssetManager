# 에이전트 스킬 만들기 (Creating Agent Skills)

이 가이드는 Gemini CLI의 기능을 확장하기 위해 직접 에이전트 스킬을 만드는 방법에 대한 개요를 제공합니다.

## 시작하기: `skill-creator` 스킬

새로운 스킬을 만드는 권장 방법은 내장된 `skill-creator` 스킬을 사용하는 것입니다. 이 기능을 사용하려면 Gemini CLI에게 새로운 스킬을 만들어 달라고 요청하세요.

**프롬프트 예시:**

> "'code-reviewer'라는 이름의 새로운 스킬을 만들어줘"

그러면 Gemini CLI는 `skill-creator`를 사용하여 스킬을 생성합니다.

1.  스킬을 위한 새 디렉토리를 생성합니다 (예: `my-new-skill/`).
2.  필요한 YAML 프론트매터(`name` 및 `description`)가 포함된 `SKILL.md` 파일을 생성합니다.
3.  표준 리소스 디렉토리인 `scripts/`, `references/`, `assets/`를 생성합니다.

## 수동으로 스킬 만들기

스킬을 직접 수동으로 만들고 싶다면:

1.  스킬을 위한 **디렉토리를 생성**합니다 (예: `my-new-skill/`).
2.  새 디렉토리 안에 **`SKILL.md` 파일**을 생성합니다.

스킬을 지원하는 추가 리소스를 추가하려면 스킬 구조를 참조하세요.

## 스킬 구조

스킬은 루트에 `SKILL.md` 파일이 포함된 디렉토리입니다.

### 폴더 구조

`SKILL.md` 파일이 유일한 필수 구성 요소이지만, 스킬의 리소스를 정리하기 위해 다음과 같은 구조를 권장합니다.

```text
my-skill/
├── SKILL.md       (필수) 지침 및 메타데이터
├── scripts/       (선택 사항) 실행 가능한 스크립트
├── references/    (선택 사항) 정적 문서
└── assets/        (선택 사항) 템플릿 및 기타 리소스
```

### `SKILL.md` 파일

`SKILL.md` 파일은 스킬의 핵심입니다. 이 파일은 메타데이터를 위해 YAML 프론트매터를 사용하고, 지침을 위해 마크다운을 사용합니다. 예시:

```markdown
---
name: code-reviewer
description:
  코드 리뷰를 위해 이 스킬을 사용합니다. 로컬 변경 사항과 원격 풀 리퀘스트(PR)를 모두 지원합니다.
---

# Code Reviewer

이 스킬은 에이전트가 철저한 코드 리뷰를 수행하도록 안내합니다.

## 워크플로우

### 1. 리뷰 대상 결정

- **원격 PR**: 사용자가 PR 번호나 URL을 제공하면 해당 원격 PR을 대상으로 합니다.
- **로컬 변경 사항**: 변경 사항이 로컬인 경우... ...
```

- **`name`**: 스킬의 고유 식별자입니다. 디렉토리 이름과 일치해야 합니다.
- **`description`**: 스킬이 수행하는 작업과 Gemini가 이 스킬을 언제 사용해야 하는지에 대한 설명입니다.
- **본문 (Body)**: 파일의 마크다운 본문에는 스킬이 활성화되었을 때 에이전트의 행동을 안내하는 지침이 포함됩니다.
