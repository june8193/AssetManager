# 기술적 요구사항: 대시보드 비활성 계좌 필터링 및 자산 왜곡 수정

## 1. 개요
현재 대시보드에서 비활성 계좌(`Legacy_Total_Account` 등)가 표시되거나, 해당 계좌의 데이터가 총 자산 합계에 반영되어 전체 수치가 왜곡되는 문제를 해결합니다.

## 2. 문제 현상
- **계좌별 현황 노출**: `is_active=False`인 계좌가 대시보드 목록에 나타남.
- **자산 수치 왜곡**: 과거 데이터 이관용 가상 계좌(`Legacy_Total_Account`)의 잔고나 트랜잭션이 합산되어 전체 자산이 실제보다 크게 표시됨.
- **연도별 통계 오류**: `get_yearly_stats`에서 모든 스냅샷을 합산하여 비활성 계좌의 데이터가 포함됨.

## 3. 변경 요구사항

### 3.1. 백엔드 (Python / FastAPI)
- **`DashboardService.get_holdings` 개선**:
    - 처음부터 `is_active=True`인 계좌의 트랜잭션만 조회하도록 쿼리 최적화.
    - 비활성 계좌에 연결된 자산은 계산에서 제외.
- **`DashboardService.get_yearly_stats` 수정**:
    - `AccountSnapshot` 조회 시 `Account.is_active == True`인 계좌의 스냅샷만 필터링하여 합산.
    - **단, 과거 특정 시점(예: 이관 전)의 통계를 위해 필요한 경우에 대한 정책 결정**: 현재는 중복 제거를 위해 비활성 계좌(특히 Legacy 계좌)를 일괄 제외하는 방향으로 구현하되, 향후 필요시 필터링 정책을 세분화할 수 있도록 구조 설계.
    - `Transaction` 기반 추가액 계산 시에도 비활성 계좌 제외.
- **`DashboardService.get_dashboard_summary` 검증**:
    - 최종적으로 반환되는 `accounts` 목록과 `total_valuation_krw`에 비활성 계좌 데이터가 포함되지 않음을 보장.

### 3.2. 데이터베이스
- `Legacy_Total_Account`(ID 1) 및 기타 비활성 계좌(`is_active=0`)의 상태 재확인 및 유지.

## 4. 기대 결과
- 대시보드 "계좌별 현황"에 실제 활성 계좌만 표시됨.
- "총 평가 자산" 및 "연도별 현황" 수치가 실제 보유 자산과 일치하게 수정됨.
- 데이터 조회 성능 향상 (불필요한 트랜잭션 처리 제외).
