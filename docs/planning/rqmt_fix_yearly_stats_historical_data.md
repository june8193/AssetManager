# 요구사양서: 연도별 현황 과거 데이터 복구 (RQMT_FIX_YEARLY_STATS_HISTORICAL_DATA)

## 1. 개요
최근 계좌별 현황에서 비활성 계좌를 제외하는 작업을 수행한 후, 연도별 현황(Yearly Status)에서 2021~2024년 등 과거 데이터가 제대로 표시되지 않는 문제가 발생함. 이는 과거에 활성 상태였으나 현재 비활성화된 계좌의 데이터가 통계에서 제외되었기 때문임.

## 2. 문제 현황
- `DashboardService.get_yearly_stats` 메서드에서 `Account.is_active == True` 필터를 적용하여, 현재 비활성 상태인 계좌의 과거 거래 내역 및 스냅샷이 통계에서 누락됨.
- `Legacy_Total_Account`와 같이 과거 데이터를 담고 있는 비활성 계좌의 데이터가 연도별 수익률 및 자산 계산에 반영되지 않음.

## 3. 요구사항
- **연도별 현황 데이터 복구**: 연도별 현황 통계 계산 시, 계좌의 현재 활성 여부와 관계없이 모든 역사적 데이터(거래 내역, 스냅샷)를 포함해야 함.
- **계좌별 현황 필터 유지**: 대시보드의 "계좌별 현황"(현재 보유 자산)에서는 비활성 계좌가 계속해서 제외되어야 함 (사용자 승인 완료된 사항).

## 4. 기술적 세부사항
- `src/backend/services/dashboard_service.py` 내의 `get_yearly_stats` 메서드 수정.
- **필터 제거**: `Transaction` 및 `AccountSnapshot` 조회 시 `Account.is_active == True` 조인 필터 제거.
- **기말 자산 계산 로직 개선**: 특정 연도의 마지막 날짜를 기준으로 합산하는 대신, **각 계좌별로 해당 연도 내 마지막 스냅샷 평가액**을 찾아 합산하여 데이터 누락 방지.
- **주석 강화**: 현재 보유 자산(Active Only)과 역사적 통계(All Data)의 필터링 정책 차이를 코드 내 한글 주석으로 명시.

## 5. 리스크 대응
- **기말 자산 데이터 누락**: 연말 이전에 계좌가 폐쇄된 경우 해당 연도 내 가장 최근 스냅샷을 사용하도록 로직 보완.
- **성능**: 대량 데이터 조회 시 성능 저하 여부 모니터링 (현재 수준에서는 문제 없음).
- **데이터 노이즈**: 거래나 잔고가 없는 무의미한 비활성 계좌는 계산 과정에서 자연스럽게 제외됨을 확인.

## 6. 검증 계획
- **단위 테스트**: `tests/test_dashboard_filter.py` (또는 신규 테스트 파일)를 작성하여 비활성 계좌의 데이터가 연도별 통계에 포함되는지 검증.
- **E2E 테스트**: 실제 대시보드 화면의 연도별 현황 테이블에 2021~2024년 데이터가 올바르게 표시되는지 확인.
