# 기술적 요구사양: 연도별 자산 통계 계산 로직 개선

## 1. 개요
대시보드의 '연도별 현황'에서 2024년도 자산 총액이 실제보다 높게 계산되는 문제를 해결합니다. 이는 과거 통합 데이터를 관리하던 계좌와 현재 개별 데이터를 관리하는 계좌가 특정 연도(2024년)에 공존하면서 발생하는 중복 합산 문제입니다.

## 2. 현황 및 문제점
### 2.1 현재 자산 계산 로직
- `DashboardService.get_yearly_stats` 메서드에서 수행.
- 각 연도별로 해당 연도에 기록된 모든 스냅샷을 검토.
- **계좌별(account_id)**로 그 연도의 **마지막 스냅샷 평가액**을 추출하여 모두 합산.
- 예: 계좌 A(10월 마지막), 계좌 B(12월 마지막)가 있다면 `A(10월) + B(12월)`을 그 연도의 기말 자산으로 간주.

### 2.2 2024년 데이터의 특이점
- **계좌 1 (Legacy_Total_Account)**: 2024년 10월 24일이 마지막 기록 (평가액: 약 1.68억).
- **계좌 2~15 (개별 계좌들)**: 2024년 12월 29일이 마지막 기록 (평가액 합계: 279,007,076원).
- **결과**: 기존 로직에 의해 `1.68억 + 2.79억`이 합산되어 약 4.47억으로 표시됨.

### 2.3 기대 결과
- 2024년 기말 자산은 12월 29일에 기록된 최신 데이터의 합계인 **279,007,076원**이어야 함.
- 10월에 기록이 중단된 계좌 1(통합 계좌)은 12월 말 시점의 합계에서 제외되어야 함.

## 3. 해결 방안
### 3.1 로직 수정 (최신 날짜 기준 합산)
- 각 연도별로 기말 자산을 계산할 때, 단순히 "계좌별 마지막 값"을 더하는 대신 **"해당 연도에서 가장 마지막으로 기록된 스냅샷 날짜(Latest Snapshot Date)"**를 찾습니다.
- **해당 날짜에 존재하는 모든 스냅샷의 평가액만 합산**하여 연도별 자산으로 정의합니다.

### 3.2 로직의 타당성 검토
- **2023년**: 마지막 기록일이 12월 4일이며, 이날은 계좌 1 데이터만 존재함 -> 12월 4일 데이터가 정상 합산됨.
- **2024년**: 마지막 기록일이 12월 29일이며, 이날은 9개 계좌 데이터가 존재함 -> 계좌 1(10월)은 제외되고 9개 계좌만 합산되어 **279,007,076원** 도출.
- **데이터 성격**: 본 프로젝트의 스냅샷은 특정 시점의 전체 자산 상태를 "동기화"하는 개념이므로, 연말의 가장 마지막 동기화 날짜를 기준으로 삼는 것이 비즈니스 로직상 타당함.

## 4. 영향 범위
