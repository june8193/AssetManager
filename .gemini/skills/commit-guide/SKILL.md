# Commit Guide Skill

이 스킬은 프로젝트의 커밋 가이드라인과 코드 리뷰 규칙을 준수하도록 설계되었습니다. 사용자가 커밋을 요청할 때 이 스킬을 활성화하여 체계적인 리뷰 및 커밋 절차를 수행합니다.

## 워크플로우

### 1. 변경 사항 분석
- `git status`와 `git diff HEAD` 명령을 실행하여 현재 스테이징되었거나 수정된 변경 사항을 모두 파악합니다.
- 만약 스테이징된 파일이 없다면, 사용자에게 커밋할 파일을 선택하도록 하거나 전체 스테이징(`git add .`) 여부를 확인합니다.

### 2. @code-reviewer 호출 및 리뷰 강제
- 파악된 변경 사항(`git diff`)을 바탕으로 `@code-reviewer` 서브에이전트를 호출합니다.
- **리뷰 요청 시 포함할 지침**:
    - `.agents/rules/code-review.md` 및 `GEMINI.md`의 규칙을 준수하는지 확인.
    - 특히 **한글 Docstring/주석**, **TDD 원칙**, **보안 및 에러 처리**를 중점적으로 검토 요청.
    - 리뷰 결과는 반드시 **한국어**로 작성하도록 요청.

### 3. 리뷰 결과 보고 및 피드백 반영
- `@code-reviewer`로부터 받은 피드백을 사용자에게 요약하여 보고합니다.
- 수정이 필요한 사항이 있다면 사용자에게 수정을 제안하고, 수정 후 다시 리뷰를 진행할지 묻습니다.
- 사용자가 리뷰 결과에 만족하고 진행을 승인할 때까지 다음 단계로 넘어가지 않습니다.

### 4. 커밋 메시지 작성 및 실행
- 리뷰가 완료되고 승인되면, 변경 사항의 의미를 가장 잘 담은 커밋 메시지를 **한국어**로 제안합니다.
- `.agents/workflows/commit.md`에 정의된 접두어(feat, fix, refactor 등)를 사용합니다.
- 사용자의 최종 승인 후 커밋을 실행합니다.

## 주의 사항
- 절대 코드 리뷰 없이 바로 커밋하지 마십시오.
- 모든 커밋 메시지와 리뷰 피드백은 한국어를 사용합니다.
- 커밋 성공 후 반드시 `git status`로 상태를 확인하여 사용자에게 알립니다.
