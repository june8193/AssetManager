# 작업 실행 스킬 (Task Executor)

이 스킬은 수립된 계획에 따라 실제 코드를 수정하고 기능을 구현하는 단계를 안내합니다. 모든 개별 작업은 서브에이전트를 통해 수행되어야 합니다.

## 실행 절차

이 스킬이 활성화되면 반드시 다음의 순서를 준수하십시오.

### 1. 관련 문서 식별 및 로드
- `work/` 디렉토리에서 실행할 대상 기능의 요구사양 문서(`rqmt_*.md`)와 작업 계획 문서(`plan_*.md`)를 찾아 내용을 읽습니다.
- 전체적인 구현 목표와 기술적 제약 사항을 파악합니다.

### 2. 개별 작업 수행 (Sub-agent Delegation)
- 작업 계획 문서에 나열된 순서대로 각 작업을 하나씩 수행합니다.
- **서브에이전트 호출**: 각 작업마다 `@generalist` 또는 작업 성격에 맞는 서브에이전트를 호출합니다.
- **컨텍스트 전달**: 서브에이전트 호출 시 다음 정보를 반드시 포함해야 합니다.
    - 현재 수행해야 할 개별 작업의 내용 및 목표
    - 참고해야 할 요구사양 문서(`rqmt_*.md`)의 핵심 내용
    - 이전 단계의 작업 결과(의존성이 있는 경우)
- **병렬 실행 금지**: 파일 충돌 및 상태 무결성을 위해, 파일을 수정하는 작업은 반드시 하나씩 순차적으로 완료한 후 다음 단계로 넘어갑니다.

### 3. 작업 결과 검증
- 각 서브에이전트의 작업이 완료되면, 해당 작업이 계획대로 이루어졌는지 확인합니다.
- 테스트 코드 실행 및 린트 체크 등을 통해 구현 결과의 품질을 검증합니다.

### 4. 진행 상황 업데이트
- 모든 작업이 완료되면 사용자에게 보고합니다.
- 만약 작업 중 계획 수정이 필요한 경우, `task-planner`를 다시 호출하거나 사용자에게 알립니다.

## 서브에이전트 위임 가이드라인 (예시)
- **코드 수정 및 구현**: `@generalist`
- **복잡한 분석 및 리팩토링**: `@codebase_investigator`
- **테스트 및 검증**: `@generalist`

## 주의 사항
- 서브에이전트에게 작업을 넘길 때 단순히 "이거 해줘"가 아니라, **"RQMT 문서에 따라 [어떠한 목표]를 달성하기 위해 [이러한 변경]을 수행해줘"**와 같이 구체적으로 지시해야 합니다.
- 프로젝트의 코딩 표준(한글 주석, TDD 등)이 준수되도록 서브에이전트에게 명확히 전달하십시오.
