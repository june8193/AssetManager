# 실행 워크플로우 (/execute)

이 워크플로우는 `plan.md`를 통해 수립된 `task_plan.md`를 실제 코드로 구현하고 검증하는 과정을 안내합니다. 모든 개발은 **TDD(Test Driven Development)** 원칙을 준수하며, 최종적으로 **E2E 테스트**를 통해 배포 가능 여부를 확인합니다.

---

## 1단계: 컨텍스트 및 계획 확인

작업을 시작하기 전, 현재 상태와 목표를 명확히 합니다.

1.  **기획 문서 로드**: `docs/planning/requirements.md`와 `docs/planning/task_plan.md`를 읽고 전체적인 문맥을 파악합니다.
2.  **작업 가시화**: `task_plan.md`에서 아직 완료되지 않은 가장 우선순위가 높은 작업을 식별합니다.
3.  **내부 작업 관리**: Antigravity 전용 `task.md` 아티팩트를 생성하거나 업데이트하여 현재 진행할 세부 단계를 기록합니다.

---

## 2단계: 서브에이전트를 통한 작업 실행 (작업별 반복)

`task_plan.md`에 정의된 개별 작업은 **서브에이전트**를 한 번 실행하여 완결합니다.

1.  **서브에이전트 실행**: 각 작업에 대해 서브에이전트를 호출합니다.
    - **Task 상세**: 수행할 작업 내용, 관련 파일 경로, TDD 준수 지침을 상세히 기술합니다.
    - **TDD 사이클 및 커밋 강제**: 서브에이전트에게 다음 과정을 완결하도록 지시합니다.
        1. `[RED]`: 실패하는 테스트 코드 작성 및 실행.
        2. `[GREEN]`: 최소한의 구현으로 테스트 통과.
        3. `[REFACTOR]`: 코드 품질 개선 및 테스트 재검증.
        4. **커밋 수행**: 모든 테스트 통과 확인 후 **`/commit` 워크플로우를 호출**하여 해당 작업 내용을 저장소에 반영.
2.  **결과 검토 및 상태 업데이트**: 서브에이전트가 반환한 리포트와 커밋 여부를 확인합니다.
    - 성공 시: `task_plan.md`와 `task.md`의 해당 작업을 `[x]`로 업데이트합니다.
    - 실패 시: 문제 원인을 파악하여 작업을 재수행하거나 기획을 수정합니다.

---

## 3단계: 전체 검증 및 E2E 테스트

개별 단위 테스트를 넘어 시스템 전체의 정합성을 확인합니다.

1.  **정적 분석**: 린트(Lint) 도구를 실행하여 코드 스타일 오류를 수정합니다.
2.  **서버 구동**: `python dev.py` 명령어를 통해 백엔드와 프론트엔드 서버를 동시에 실행합니다. (배경 실행)
3.  **E2E 테스트 실행**: 
  - 브라우저 도구를 사용하여 실제 시나리오대로 기능이 작동하는지 확인합니다.
  - `GEMINI.md`의 E2E 테스트 규칙을 준수합니다.
4.  **서버 종료**: 검증이 완료되면 구동 중인 프로세스를 안전하게 종료합니다.

---

## 4단계: 마무리 및 최종 검증

1.  **전체 테스트 재실행**: 모든 작업이 완료된 후 전체 테스트 수트를 다시 한 번 실행하여 사이드 이펙트가 없는지 확인합니다.
2.  **워크스루 작성**: 구현 내용과 테스트 결과를 요약하여 `walkthrough.md`를 작성합니다.
3.  **종료 보고**: 사용자에게 모든 작업의 완료 및 커밋 상태를 보고합니다.
