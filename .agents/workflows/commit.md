# 커밋 워크플로우

이 워크플로우는 프로젝트의 변경사항을 커밋할 때 준수해야 하는 단계별 가이드라인입니다.

## 1. 변경사항 검토 및 스테이징
- 변경된 파일들을 확인하고 커밋할 내용을 결정합니다.
- `git add`를 사용하여 변경사항을 스테이징 영역에 추가합니다.

## 2. 코드 리뷰 수행
- 커밋하기 전에는 반드시 코드 리뷰를 거쳐야 합니다.
- 코드 리뷰는 `.agents/rules/code-review.md`의 규칙에 따라 수행합니다.
- 다음 항목들을 중점적으로 확인합니다:
    - 가독성 및 유지보수성
    - 에러 처리
    - 보안 및 성능
    - **한글로 작성된 Docstring 및 주석** (Google 스타일)

## 3. 커밋 메시지 작성
- 커밋 메시지는 반드시 **한국어**로 작성합니다.
- 변경 사항의 성격에 따라 적절한 접두어(feat, fix, refactor, style 등)를 사용하기를 권장합니다.

## 4. 커밋 실행
- 스테이징된 내용과 메시지를 최종 확인 후 커밋을 실행합니다.
