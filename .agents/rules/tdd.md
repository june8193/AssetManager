
# TDD (Test Driven Development) 규칙

이 프로젝트는 TDD 방식으로 개발합니다. 새로운 기능을 구현하거나 기존 코드를 수정할 때 반드시 아래 절차를 따릅니다.

## 개발 절차

1. **Red** — 실패하는 테스트를 먼저 작성합니다.
2. **Green** — 테스트를 통과하는 최소한의 코드를 작성합니다.
3. **Refactor** — 테스트가 통과하는 상태를 유지하면서 코드를 개선합니다.

## 공통 규칙

- 프로덕션 코드를 작성하기 전에 반드시 해당 기능의 테스트를 먼저 작성합니다.
- 모든 공개 함수/메서드는 최소 하나 이상의 테스트를 가져야 합니다.
- 버그 수정 시에도 해당 버그를 재현하는 테스트를 먼저 작성한 후 수정합니다.

## 백엔드 (Python)

- 테스트 프레임워크: `pytest`
- 테스트 파일 위치: `tests/` 디렉토리
- 테스트 파일 네이밍: `test_` 접두사 (예: `test_api.py`)

## 프론트엔드 (React)

- 테스트 러너: `Vitest`
- 컴포넌트 테스트: `React Testing Library`
- 테스트 파일 위치: 컴포넌트와 같은 디렉토리에 배치
- 테스트 파일 네이밍: `.test.jsx` 접미사 (예: `Dashboard.test.jsx`)
