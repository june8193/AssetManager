# 코드 리뷰 규칙

코드 리뷰를 진행할 때 다음 사항들을 확인하고 피드백을 제공합니다.

## 일반 원칙
- **건설적인 피드백**: 문제점뿐만 아니라 개선을 위한 구체적인 대안이나 레퍼런스를 함께 제시합니다.
- **한국어 사용**: 모든 코드 리뷰 코멘트와 피드백은 한국어로 작성합니다.

## 코드 품질 가이드라인
1. **가독성 및 유지보수성**: 변수, 함수의 네이밍이 명확한지, 코드가 중복되지 않고 간결한지 확인합니다.
2. **에러 처리**: 적절한 예외 처리(Exception Handling)가 되어 있는지, 예기치 않은 오류 방어가 제대로 되었는지 검토합니다.
3. **보안 및 성능**: 불필요한 연산이나 런타임 성능 저하를 일으킬 수 있는 코드가 없는지, 민감한 정보가 하드코딩 되어 있지 않은지 확인합니다.
4. **문서화 및 동기화**: 
   - 모든 주요 기능과 모듈에 `GEMINI.md`의 공통 규칙에 따른 Google 방식의 한글 주석과 Docstring이 제대로 작성되어 있는지 검토합니다.
   - 주요 코드 변경사항이 docstirng, 주석과도 일치하는지 확인합니다.
5. **테스트 용이성**: 작성된 코드가 독립적으로 테스트 가능한 구조인지 확인합니다.
